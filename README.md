## 移动 CMI 线路到底好在哪，和普通线路差多少

### 什么是 CMI，为什么它对国内用户重要

中国移动在国内的用户基数极大，但很多人不知道，移动的国际出口走的是自家的 CMI 骨干网，而不是和电信、联通共用的公网通道。这意味着：

- 去程（国内 → 海外）：移动用户访问海外服务器，流量从 CMI 出境
- 回程（海外 → 国内）：服务器响应数据，如果走 CMI 回来，就能避开公网拥堵

真正的移动 CMI 优化，指的是**双向都走 CMI 骨干**，而不只是单向。

### 晚高峰才是真正的考验

我自己测过不少机器，白天延迟看起来都差不多，差距在晚上 8点到 11 点。普通线路这个时段丢包率能到 5%-15%，CMI 优化线路通常能压在 1% 以内。

具体感受：同一台机器，白天跑 SSH 很流畅，晚高峰开始之后，普通线路的终端输入会有明显卡顿感，CMI 线路基本感觉不到差异。

## 常见问题

### 移动 CMI 和 CN2 GIA 有什么区别，我该选哪个？

CN2 GIA 是电信的优化线路，CMI 是移动的。如果你的主要用户或你自己用的是移动宽带，CMI 回程更直接。DMIT 的 Pro 系列同时包含两者，不用二选一。如果只用电信宽带，CN2 GIA 单独就够；移动用户多的场景，CMI 是关键。

### DMIT 的香港节点和洛杉矶节点怎么选？

延迟优先选香港，价格和流量性价比选洛杉矶。香港物理距离近，延迟通常在 30-50ms；洛杉矶延迟 160-180ms，但套餐流量更大、价格相对低。业务对实时性要求高就选香港，一般用途洛杉矶够用。

### 晚高峰 CMI 线路真的不卡吗？

不是"完全不卡"，而是比普通线路稳定很多。CMI 走专用骨干，受公网拥堵影响小，但极端情况下（比如大规模网络事件）也会有波动。长期用下来，晚高峰丢包率明显低于普通线路，这是实际感受。
